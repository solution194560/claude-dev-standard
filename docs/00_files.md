# 00. 파일별 상세 설명서 (File Guide)

이 킷에 들어 있는 **모든 파일**의 용도·내용·읽는 시점을 한 곳에 정리한 설명서입니다.
"이 파일이 뭐지?" 싶을 때 여기로 오세요.

```
claude-dev-standard/
├── README.md                  ← 킷 소개 + 지도 (첫 진입점)
├── docs/                      ← 📖 읽는 곳 (복사하지 않음)
│   ├── 00_files.md            ← 지금 이 문서
│   ├── 01_quickstart.md       ← 10분 시작 가이드
│   ├── 02_process.md          ← 5단계 게이트 상세
│   ├── 03_agents.md           ← 에이전트 5종 + Codex 연동(기본)
│   ├── 04_rules.md            ← 개발 규칙과 그 이유
│   ├── 05_session.md          ← 세션 이어가기
│   └── 06_cost.md             ← 비용 통제
└── templates/                 ← 📦 내 프로젝트로 복사하는 곳
    ├── CLAUDE.md.template     ← → 내 프로젝트의 CLAUDE.md
    └── .claude/
        ├── settings.json.example  ← → .claude/settings.json
        └── agents/                ← → 그대로 복사 (5종)
            ├── plan-writer.md
            ├── plan-reviewer.md
            ├── implementer.md
            ├── impl-verifier.md
            └── final-tester.md
```

## 📖 docs/ — 읽는 문서

### README.md (킷 루트)

| 항목 | 내용 |
|---|---|
| 한 줄 | 킷의 첫 진입점 — 무엇을 주고 무엇을 안 주는지, 전체 지도 |
| 언제 읽나 | 킷을 처음 받았을 때 (5분) |

**주요 내용**: 패키지 지도(mermaid — docs는 읽고 templates는 복사),
5단계 게이트 흐름도와 단계·산출물·판정 표, 시작하기 3단계,
문서 목차, 핵심 원칙 5개 요약(설계 먼저 / 게이트 통과 전 진행 금지 /
실쓰기 금지선 / 기록 / 교차 검증 기본).

### 01_quickstart.md

| 항목 | 내용 |
|---|---|
| 한 줄 | 복사 → 치환 → 첫 실행까지 10분 안에 끝내는 절차서 |
| 언제 읽나 | 새 프로젝트에 킷을 적용할 때 (필독, 순서대로 따라하기) |

**주요 내용**:
- **사전 준비 표** — Claude Code 설치 확인, 프로젝트 폴더, Codex CLI(권장 — 없어도 동작)
- **① 템플릿 복사** — PowerShell/bash 복사 명령 (그대로 실행 가능)
- **② CLAUDE.md 채우기** — `{{ }}` 치환 지점 6개 표 (프로젝트명·개요·실행 명령·
  테스트 명령·위험 작업 목록·외부 점검 도구). **가장 중요한 단계** — 에이전트들이
  이 프로필을 읽고 동작하므로 여기만 채우면 에이전트 파일은 수정 불필요
- **③ 첫 실행** — 5단계를 호출하는 대화 예시 (계획→점검→구현→검증→최종)
- **FAQ** — 에이전트 파일 수정 여부, 단건 수정 경량 경로, settings.json 용도,
  세션 중단·재개 방법, 토큰 비용 걱정

### 02_process.md

| 항목 | 내용 |
|---|---|
| 한 줄 | 5단계 게이트의 단계별 규칙·산출물 명명·생략 기준의 정본 |
| 언제 읽나 | 첫 개발 시작 전 + 게이트 판정이 헷갈릴 때 |

**주요 내용**:
- **전체 흐름 상태도** — REVISE/FAIL/BLOCKED 시 어느 단계로 돌아가는지
- **단계별 규칙** — 각 단계의 산출물·착수 조건·금지:
  - 1 계획: 자기완결적 문서(대화 맥락 의존 금지), 필수 포함 항목, 코드 작성 금지
  - 2 점검(게이트①): 점검 관점 5가지(사실 대조·정책 충돌·완결성·위험·테스트), APPROVE까지 반복
  - 3 구현: 착수 조건 = APPROVE, Phase 범위 준수, 편차 보고, 테스트 추가 의무
  - 4 검증(게이트②): 테스트 직접 실행, 테스트 실패 = 무조건 FAIL, 안전 규칙 감사
  - 5 최종(게이트③): 실데이터 e2e, 쓰기는 드라이런까지, DONE 시 CHANGELOG 갱신
- **산출물 명명 규칙** — `PLAN_<주제>.md` / `_REVIEW` / `_VERIFY_<phase>` / `_FINAL_<phase>`
- **5단계 생략 기준** — 판단 플로차트 + 경량 경로(단건 수정)에서도 생략 불가인 4가지

### 03_agents.md

| 항목 | 내용 |
|---|---|
| 한 줄 | 에이전트 5종의 역할·호출법·모델 교체 + **Codex 연동(기본 권장)·미사용 폴백** |
| 언제 읽나 | 에이전트를 처음 부를 때, Codex를 연결/해제할 때, 모델을 바꿀 때 |

**주요 내용**:
- **한눈에 보기 표** — 5종의 모델·역할·산출물·소스 수정 가능 여부(심판과 선수의 분리)
- **호출 방법** — 자연어 호출 예시 5개
- **에이전트 파일 구조** — frontmatter(name/description/model/tools) 설명,
  모델 교체는 `model:` 한 줄, 도구 제한은 안전장치의 일부일 뿐(Write·Bash 로 우회 가능)
- **Codex 연동 (기본 권장)** — 최초 설정 3단계(설치 확인 → Windows 전용 `[windows]
  sandbox = "unelevated"` 설정 → CLAUDE.md 프로필에 정규 명령 기재). macOS·Linux와
  Windows 명령을 각각 병기. 주의사항, 플랫폼별 문제 해결 표
- **Codex 없이 쓸 때** — 미설정(`없음` 기재)·실행 실패(자동 폴백)·이번만 제외, 3경로와 한계

### 04_rules.md

| 항목 | 내용 |
|---|---|
| 한 줄 | 개발 규칙의 정본 — 각 규칙이 **왜** 있는지 이유까지 |
| 언제 읽나 | 킷 적용 초기 (필독) + 규칙이 과하다고 느껴질 때 (Why 절 참조) |

**주요 내용**:
- **규칙 지도(mindmap)** — 설계 먼저 / 안전 금지선 / 품질 / 기록 4축
- **1. 설계 먼저** — 방안 2~3개 비교 → 확인 → 구현. Why: 설계 검토 5분이 재작업 5시간을 막음. 예외 2가지
- **2. 안전 금지선** — 실쓰기는 드라이런까지·게이트 우회 금지·시크릿은 .env. Why: 사고는 "실수로 운영에 썼다"에서 나옴
- **3. 품질 규칙** — 결정적 로직 오프라인 테스트, 검증자 직접 실행, 정책 변경 영향 범위, 오탐 수정 시 재현 테스트
- **4. 기록 규칙** — CHANGELOG 맨 위, CLAUDE.md 얇게, 판정은 보고서로
- **5. 협업 권장** — settings 공유/개인 구분, .gitignore 최소, 보고서 커밋

### CLAUDE.md.template → 내 프로젝트의 `CLAUDE.md`

| 항목 | 내용 |
|---|---|
| 한 줄 | Claude Code가 **매 대화마다 자동으로 읽는** 프로젝트 지침의 뼈대 |
| 손댈 곳 | `{{ }}` 치환 지점 전부 (특히 §0 프로필 — 필수) |

**섹션 구성**:
- **§0 프로젝트 프로필** — 개요·실행 명령·테스트 명령·산출물 경로·외부 점검 도구
  + **위험 작업 목록**(에이전트 실쓰기 금지 대상). **에이전트 5종이 전부 이 절을
  참조**하므로 반드시 채워야 함
- **§1 작업 원칙** — 설계 먼저(2~3안 비교 → 확인 → 구현) 규칙 본문
- **§2 변경 이력·세션 기록** — CHANGELOG 분리 규칙 + SESSION.md 체크포인트 규칙
- **§3 실행** — 실행/테스트 명령 (치환)
- **§4 개발 프로세스** — 5단계 게이트 표 + 외부 점검 도구·실쓰기 금지·경량 경로 규칙
- **§5 정책 요약** — 프로젝트 도메인 규칙 자리 (없으면 삭제)

### .claude/settings.json.example → `.claude/settings.json`

Claude Code 도구 권한 설정. `allow`는 테스트 실행 정도만 자동 허용하고, `deny`는
킷의 안전 금지선을 **강제**한다(실쓰기 `--apply`·`git push`, 게이트 우회 `--yes`,
시크릿 `.env`). `deny`가 `allow`보다 먼저 평가되므로 `Read(**)`가 있어도 `.env`는 막힌다.
**CLAUDE.md §0 위험 작업 목록과 짝을 맞춰 갱신할 것** — 한쪽만 고치면 강제가 사라진다.
한계는 [04_rules.md](04_rules.md#지시문과-강제는-다르다) 참조.
팀 공유는 `settings.json`, 개인 전용은 `settings.local.json`(git 제외).

### .claude/agents/ — 서브에이전트 5종 (수정 없이 복사)

공통 구조: frontmatter(name·description·model·tools) + 지시문.
**프로젝트 고유 정보는 전부 CLAUDE.md §0에서 읽으므로 파일 수정 불필요** —
`model:`(모델 교체) 외에는 손대지 않는 것을 권장. 5종 모두 공통 금지 사항은
CLAUDE.md §0 "위험 작업 목록"의 실쓰기 실행이다.

> 아래 표의 **도구 목록은 사본**입니다. 정본은 각 에이전트 파일의 `tools:` frontmatter이며,
> 어긋나면 그쪽을 믿으세요.

#### plan-writer.md — 1/5 계획 수립

| 항목 | 내용 |
|---|---|
| 모델 / 도구 | Opus / Read·Grep·Glob·Write·Edit (실행 도구 없음 — 문서만) |
| 산출물 | `PLAN_<주제>.md` (프로젝트 루트) |
| 핵심 규칙 | **자기완결적 문서** — 이 문서만 읽고 점검·구현이 가능해야 함(대화 맥락 의존 금지). "현재 코드 상태"는 소스를 실제 조사해 사실 기반으로 작성(가짜 전제는 점검에서 결함 처리). Phase 분할·테스트 기준(오프라인/실데이터 구분)·"점검 시 확인 요청 사항" 필수 포함 |
| 금지 | 소스 코드 작성/수정, 기존 계획 문서 무단 삭제 |

#### plan-reviewer.md — 2/5 계획 점검 (게이트 ①)

| 항목 | 내용 |
|---|---|
| 모델 / 도구 | Opus / Read·Grep·Glob·Write·Bash (`Edit` 없음 — 소스 수정 금지는 지시문으로 지킴) |
| 산출물 | `<PLAN>_REVIEW.md` — **APPROVE / REVISE** |
| 핵심 규칙 | CLAUDE.md §0 "외부 점검 도구" 설정 시 **그 도구(Codex) 결과가 판정 기준**(지시문 파일 → stdin 파이프, 비판적 검토 역할 지시 주입). 미설정/실패 시 직접 점검 폴백(사유 명기). 점검 관점: 사실 대조·정책 정합성·완결성·위험·테스트 검증 가능성 + 계획의 "확인 요청 사항"에 전부 답변 |
| 금지 | 소스·계획 문서 수정 (보고서만) |

#### implementer.md — 3/5 구현

| 항목 | 내용 |
|---|---|
| 모델 / 도구 | Sonnet / Read·Grep·Glob·**Edit·Write**·Bash (5종 중 유일하게 소스 수정 가능) |
| 산출물 | 코드 + CHANGELOG 기록 (Phase 단위) |
| 핵심 규칙 | **착수 조건: REVIEW가 APPROVE** (아니면 보고 후 종료). 지시받은 Phase만 구현(선행 금지). 계획과 다르게 해야 하면 임의 변경 대신 편차 보고. 새 결정적 로직에 오프라인 테스트 추가 의무. 완료 전 자체 검증: 컴파일 + 전체 테스트 PASS + 수용 기준 확인 |
| 금지 | 실쓰기 게이트(드라이런 기본값·확인 프롬프트) 우회 코드 작성, 실쓰기 실행 |

#### impl-verifier.md — 4/5 구현 검증 (게이트 ②)

| 항목 | 내용 |
|---|---|
| 모델 / 도구 | Opus / Read·Grep·Glob·Write·Bash (`Edit` 없음) |
| 산출물 | `<PLAN>_VERIFY_<phase>.md` — **PASS / FAIL** |
| 핵심 규칙 | 코드 리뷰는 외부 도구(Codex) 우선·결과가 판정 기준, 실패 시 직접 폴백. **테스트는 도구와 무관하게 항상 직접 실행** — 구현자의 "통과" 보고 불신, **테스트 실패 = 외부 리뷰가 PASS여도 최종 FAIL**. 안전 규칙 감사(실쓰기 게이트 우회·정책 파일 광범위 수정·의존성 임의 변경 = 즉시 FAIL). scope creep(계획 밖 무단 변경)도 결함 |
| 금지 | 소스 수정 (결함은 보고만 — 수정은 implementer 몫) |

#### final-tester.md — 5/5 최종 완료 테스트 (게이트 ③)

| 항목 | 내용 |

| 산출물 | `<PLAN>_FINAL_<phase>.md` — **DONE / BLOCKED** |
