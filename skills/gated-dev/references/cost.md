# 06. 비용 통제 (Cost Control)

Claude Code 비용은 대부분 **토큰**(모델이 읽고 쓰는 텍스트 양)에서 나옵니다.
초급자가 기억할 원칙은 하나입니다:

> **비싼 모델은 판단에만 쓰고, 반복·기계적인 일은 싼 경로로 돌린다.**

## 1. 모델 티어 분배 — 킷의 기본 설계

5단계 게이트는 이미 단계별로 모델을 나눠 쓰도록 설계돼 있습니다.
**어느 단계가 어느 모델인지는 [agents.md](agents.md#한눈에-보기)의 표가 정본**이고,
여기서는 그렇게 나눈 *이유*만 설명합니다.

- **판단이 필요한 곳(1 계획 · 2 점검 · 4 검증)은 상위 모델** — 여기서 아끼면
  재작업으로 더 나갑니다.
- **실행 중심인 곳(3 구현 · 5 테스트)은 중간 티어로 충분** — 판단보다 절차를 따르는 일입니다.
- **2·4단계에 외부 도구를 붙이면 그 리뷰 분량만큼 Claude 토큰을 아낍니다** —
  교차 검증 효과는 덤 ([agents.md](agents.md)).

모델 교체는 에이전트 파일 frontmatter의 `model:` 한 줄입니다 ([agents.md](agents.md#에이전트-파일-구조-수정하고-싶을-때)).

## 2. 매 턴 나가는 고정 비용 줄이기

**CLAUDE.md는 매 대화 턴마다 자동 로드**됩니다. 여기 적힌 모든 글자가
매번 토큰으로 계산됩니다.

- 작업 기록·과거 이력은 **CHANGELOG.md로** (필요할 때만 읽음) — 이 킷의
  기본 규칙 ([rules.md](rules.md#4-기록-규칙))
- 현재 유효한 규칙·정책만 CLAUDE.md에 남기기
- 긴 표·상세 절차는 별도 문서로 빼고 CLAUDE.md에는 링크만

## 3. 대화 습관 — 티끌이 가장 큽니다

| 습관 | 비싼 방식 ❌ | 싼 방식 ✅ |
|---|---|---|
| 파일 읽히기 | "lib/staff.py 봐줘" (통째로) | "lib/staff.py 의 match_rule 함수만 봐줘" |
| 주제 전환 | 한 세션에서 기능 A → 버그 B → 문서 C | 주제 바뀌면 체크포인트 → 새 세션 ([session.md](session.md)) |
| 같은 설명 반복 | 매 세션 배경 설명 | 결정 사항을 CLAUDE.md/CHANGELOG에 남겨 다시 설명 안 하기 |
| 탐색 | "프로젝트 전체 구조 분석해줘" (습관적) | 필요한 질문을 구체적으로 — "담당자 추천이 어느 파일이야?" |

세션이 길어질수록 **매 턴** 이전 대화 전체가 다시 처리됩니다 — 긴 세션의
마지막 10턴이 짧은 세션 전체보다 비쌀 수 있습니다. 주제별 새 세션이 답입니다.

## 4. LLM 없이 도는 경로를 만들어 활용하기

반복 실행되는 판정·검증 로직은 **규칙 기반(코드) 경로**를 함께 만들어두면
LLM 호출 없이(=무료로) 재실행할 수 있습니다.

- 예: 키워드 판정 규칙을 config + 파이썬으로 구현해두면, 규칙을 고칠 때마다
  LLM 재실행 없이 **재판정 스크립트만 돌려** 빠르게 피드백을 받습니다.
  LLM 은 최초 분류에만 쓰고, 규칙 튜닝은 무료 경로로 반복하는 구조입니다.
- 이 킷의 품질 규칙("결정적 로직에는 오프라인 테스트")과 같은 방향입니다 —
  pytest 는 공짜고, LLM 재검증은 비쌉니다.

## 5. 외부 점검 도구 = 비용 분산

2·4단계 점검을 Codex CLI로 돌리면 그 리뷰 분량만큼 Claude 토큰을 쓰지 않습니다
(Codex는 별도 요금 체계). 교차 검증 품질 + 비용 분산을 동시에 얻는 구조라
이 킷은 연동을 **기본으로 권장**합니다 — 설정법은 [agents.md](agents.md#외부-점검-도구-연동-기본-권장--codex-cli).

## 요약 카드

```
① 판단 = 상위 모델, 실행 = 중간 모델, 리뷰 = 외부 도구(Codex)
② CLAUDE.md 는 얇게 — 기록은 CHANGELOG 로
③ 파일은 범위 지정해서 읽히기, 주제 바뀌면 새 세션
④ 반복 판정은 규칙 코드로 — pytest 는 공짜
```
