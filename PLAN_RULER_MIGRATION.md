## 1) 배경 / 목표

### 배경
1차 테스트(PLAN_RULER)로 Ruler 변환 메커니즘이 검증됐다 — 재흡수 없음, 반복 안정,
revert 정상, §0 보존, Claude·Codex 안전룰 의미 일치. 그러나 현재 공통 규칙(안전·프로세스·증거)은
`.ruler/` 와 `CLAUDE.md` 양쪽에 **중복 존재**하고, 생성물은 본 저장소에 없어서 Claude 세션은
여전히 `CLAUDE.md` 사본만 읽는다. 이번 과제는 그 중복을 제거해 **`.ruler/` 를 공통 규칙의
단일 정본**으로 만들고, `CLAUDE.md` 는 프로젝트 고유 내용만 남긴 뒤 `@RULER_CLAUDE.md`
임포트로 공통 규칙을 로드하게 한다.

### 목표
1. `CLAUDE.md` 의 공통 규칙 중복분(§4 일부·안전 문구)을 제거해 얇게 만든다 — **중복 제거가 목적**.
2. `CLAUDE.md` 맨 아래 `@RULER_CLAUDE.md` 임포트 추가(본 저장소 개발 전용).
3. **생성물 유지 A안(사용자 확정)**: `RULER_CLAUDE.md`·`AGENTS.md` 를 본 저장소에 **커밋**한다.
   `.ruler/` 수정 시 **사람이 직접** 본 저장소에서 `ruler apply` 를 실행해 재생성 후 함께 커밋.
4. 정본(`.ruler/`)↔생성물 불일치를 막는 검사 수단을 도입한다.

### 비목표
- 5단계 게이트·플러그인 구조 변경, `agents/*.md`·`skills/gated-dev/**` 수정
- MCP 변환·스킬 배포·서브에이전트 변환 활성화, 대상 확장(Cursor 등)
- `templates/` 수정 일체(임포트의 templates 배포 금지 — 1차 점검 확정 사항)
- deny 규칙 보강(`Bash(*ruler apply*)` 추가) — 1차의 후속 검토 항목이나 이번 범위 밖
  (에이전트의 ruler 실쓰기 금지는 절차 규칙으로 유지, §6)

### 1차 원칙의 개정 (명시)
이 과제는 1차 테스트의 두 원칙을 **의도적으로 개정**한다(개정 이력에도 기재).
| 1차 원칙 (PLAN_RULER) | 이관 후 (이 PLAN) |
|---|---|
| 생성물 비생성·비커밋 (본 저장소) | **생성물 커밋** — `RULER_CLAUDE.md`·`AGENTS.md` 를 git 추적 |
| 본 저장소에서 ruler 실쓰기 금지(복제본 한정) | **사람에 한해** 본 저장소 apply 허용(절차 §3.5). **에이전트 실쓰기 금지는 그대로 유지** |

### 3.2 `ruler.toml` 변경 (쟁점 1 — gitignore 충돌 제거)

```toml
# Ruler 대상·출력 설정 (정본 이관 운영본)
# 이 파일은 .md 가 아니므로 규칙 이어붙이기 대상에서 제외된다.

default_agents = ["claude", "codex"]

[gitignore]
enabled = false           # A안(생성물 커밋)과 충돌하므로 비활성 — 생성물을 git 이 추적해야 한다

[backup]
enabled = false           # 생성물이 git 추적되므로 복원 수단은 git — .bak 노이즈 제거 (점검 답변 1 로 승인 확정)

[mcp]
enabled = false           # MCP 변환 비활성 (유지)

[agents.claude]
enabled = true
output_path = "RULER_CLAUDE.md"

[agents.codex]
enabled = true
output_path = "AGENTS.md"
```

- **`[gitignore] enabled = false` 는 필수** — 1차 설정(true)은 생성물을 `.gitignore` 에 자동
  등록하므로 A안(생성물 커밋)과 정면 충돌한다.
- **변경 영향(1차 테스트 시나리오 기준과의 차이)**: ① 시나리오 1(dry-run) — "Would update
  .gitignore" 출력이 사라짐 ② 시나리오 2 — apply 후 기대가 ` M .gitignore` 1건이 아니라
  `?? RULER_CLAUDE.md`·`?? AGENTS.md`(git 추적 전) 2건으로 바뀜 ③ 시나리오 11(revert) —
  ".gitignore 블록 원복" 항목 자체가 소멸(블록을 만들지 않으므로). 이 차이를
  `ruler-test/run-ruler-test.sh` 의 기대값에 반영한다(§3.6).
- `[backup] enabled = false` 는 **점검 답변 1 로 승인 확정**됐다 — 생성물이 커밋되면 git 이
  곧 백업·복원 수단이고, apply 재생성 시 `.bak` 파일이 워킹트리에 남는 노이즈를 제거한다.
  (대안인 true 유지 + `.gitignore` `*.bak` 등록은 "`.gitignore` 무변경" 수용 기준과 상충해 기각.)
  이 확정은 [A-1]과 연동된다 — 스크립트의 `--backup` 플래그도 함께 정리(§4 Phase 1).

`.ruler/` 를 수정할 때마다 **사람(사용자)이 본 저장소 루트에서** 다음 순서로 실행한다.


  생성이 일어나지 않게 하는 이중 방어. **스크립트(§4 [A-1])와 동일한 플래그 세트로 통일**한다
  (2차 점검 결함 2 정리).
- **플래그 실재 유보([B-1] — 2단 확정)**: `--no-gitignore`·`--no-backup` 의 실재는 아직 실물
  미검증이다(1차 로그의 `--help` 는 최상위 명령만 출력 — apply 서브커맨드 옵션 미캡처).
  Phase 1 검증에서 **`apply --help` 캡처 → 플래그 실재 확정 → 이 절차·스크립트 명령을 최종
  fix** 순으로 처리한다(§4 Phase 1). 특정 플래그가 없으면 해당 플래그를 명령에서 제거하고
  `ruler.toml` 설정만 신뢰하되(예: `--no-backup` 부재 시 `[backup]=false` 신뢰), 절차 4 의

## 6) 제약 · 주의
- **§0 위험 작업 유지**: `git push` 금지, `[y/N]` y 금지 — 사람 절차(§3.5)에서도 push 는 별도.
- **에이전트의 ruler 실쓰기 금지 유지**: 본 저장소·복제본 불문, 에이전트는 `ruler apply`/
  `revert` 를 실행하지 않는다(dry-run 도 이번 과제에선 불필요 — 사람 절차에 포함).
  check-sync.sh 는 읽기 전용이라 에이전트 실행 가능.
- **버전 고정 유지**: `npx @intellectronica/ruler@0.3.44`. 버전 변경 시 check-sync.sh 의 출력
  형식 가정(헤더·Source 주석)을 재검증할 것(§3.7 (b) 단점 대응).
- **`--no-skills`·`--no-mcp` 유지, `--subagents` 금지** — 사람 apply 명령(§3.5)에 명시됨.
- **`templates/.gitignore.example` 이름 변경 금지**(§5 정책) — 이번 과제에서 templates/ 는
  일절 건드리지 않는다(임포트 배포 금지 포함).
- **CLAUDE.md 수정은 Phase 3 한정** — 계획·점검 단계에서는 무수정. 수정 범위는 §3.1 표가 정본.
- **순서 강제**: 임포트 추가(Phase 3)는 생성물 커밋(Phase 2) 이후에만 — import 깨짐 방지.
- **정본·생성물 혼동 방지**: `RULER_CLAUDE.md`·`AGENTS.md` 를 직접 수정하지 않는다.
  내용 변경은 반드시 `.ruler/` → 사람 apply 경로로만.
